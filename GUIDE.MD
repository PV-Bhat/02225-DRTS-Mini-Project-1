### 2. Generating Task Sets (Single Core, DM/EDF Compatible)

Since the project is **single-core** with **deadlines ‚Č§ periods**, use these flags:

```bash
# Generate 10 task sets, 5 tasks each, 60% utilization, UUniFast algorithm, single core
python task_generator.py -n 5 -s 10 -u 60 -g 1 -p 1 --npe 1 -r
```

| Flag | Meaning | Recommended |
|------|---------|-------------|
| `-n N` | Tasks per set | 5‚Äď10 |
| `-s N` | Number of sets | 10‚Äď50 |
| `-u N` | Total utilization (%) | Try 30, 50, 70, 90 |
| `-g N` | Generator: `1`=UUniFast, `2`=Emberson | `1` (UUniFast gives exact utilization) |
| `--npe 1` | **Single core** | **Required for your project** |
| `-r` | Round WCET/BCET to integers | Recommended |

> **Important:** By default, `Deadline = Period`, so the constraint Di ‚Č§ Ti is already satisfied.

The output CSV format is:
```

T0,   0,      31,   44,   1000,   1000,     0
T1,   0,      87,   123,  2000,   2000,     0
...
```

### 4. What You Need to Implement Yourself

The tools give you **task sets** and **job lists**. You still need to write the analysis yourself:

#### A) DM Worst-Case Response Time (Buttazzo ¬ß4.5.2)

Iterative fixed-point calculation for each task ŌĄŠĶĘ (sorted by deadline):

```
R_i^(0) = C_i
R_i^(k+1) = C_i + ő£_{j‚ąąhp(i)} ‚ĆąR_i^(k) / T_j‚ĆČ ¬∑ C_j

Stop when R_i^(k+1) == R_i^(k)  ‚Üí  WCRT_i = R_i
If R_i > D_i  ‚Üí  task is NOT schedulable
```

where `hp(i)` = set of tasks with higher priority than ŌĄŠĶĘ (shorter deadline under DM).

#### B) EDF WCRT ‚ÄĒ Analytical Hyperperiod Method (from your assignment doc)

```
1. H = lcm(T1, T2, ..., Tn)
2. Generate all jobs: ŌĄŠĶĘ releases job k at time r_{i,k} = k¬∑TŠĶĘ, for k¬∑TŠĶĘ < H
3. Simulate EDF from t=0 to t=H:
   - Every job executes exactly WCET (CŠĶĘ)
   - At each instant, schedule the ready job with earliest absolute deadline d_{i,k} = r_{i,k} + DŠĶĘ
4. Record finish time f_{i,k} for each job
5. R_{i,k} = f_{i,k} ‚ąí r_{i,k}
6. WCRT_i = max_k { R_{i,k} }
```

#### C) EDF Schedulability Check (Processor Demand, Buttazzo ¬ß4.6.1)

For constrained deadlines (Di ‚Č§ Ti), check:

```
U = ő£ CŠĶĘ/TŠĶĘ ‚Č§ 1    (necessary condition)

For all scheduling points t in [0, H]:
  dbf(0,t) = ő£ŠĶĘ ‚Ćä(t + TŠĶĘ ‚ąí DŠĶĘ) / TŠĶĘ‚Ćč ¬∑ CŠĶĘ  ‚Č§ t
```

### 5. Comparison Strategy

**DM vs EDF:**
- Generate task sets at increasing utilizations (e.g., 50%, 60%, 70%, 80%, 90%, 95%)
- For each set, check schedulability under both algorithms
- Plot: % of schedulable sets vs utilization ‚Üí EDF should dominate at high utilization
- Compare WCRTs: under DM, high-priority tasks get short WCRTs; under EDF the distribution differs

**Analytical vs Simulation:**
- In your simulator, draw each job's execution time uniformly from `[BCET, WCET]`
- Run the simulation many times (e.g., 1000 iterations √ó hyperperiod)
- Plot the distribution of observed response times vs the analytical WCRT
- The WCRT should be an **upper bound** ‚ÄĒ observed RTs should never exceed it

### 7. Task Set CSV Format Quick Reference

| Column | Meaning |
|--------|---------|
| `Name` | Task ID (T0, T1, ...) |
| `Jitter` | Release jitter (0 = synchronous release) |
| `BCET` | Best-case execution time |
| `WCET` | Worst-case execution time CŠĶĘ |
| `Period` | Task period TŠĶĘ |
| `Deadline` | Relative deadline DŠĶĘ (= TŠĶĘ by default) |
| `PE` | Processing element (0 for single core) |

For your analysis:
- Use **WCET** as `CŠĶĘ` for analytical WCRTs
- Use random value in **[BCET, WCET]** for simulation
